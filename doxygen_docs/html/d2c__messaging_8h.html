Submits the message to messaging utility queue. If the message for specified type already exist, it will be replaced by the latest message. 

   IMPORTANT: The implementation of @p responseCallback, @p completedCallback, and @p statusChangedCallback MUST NOT
   call any ADUC_D2C_* functions. Otherwise, a dead-lock may occurs.
Parameters
typeThe message type.
cloudServiceHandleAn opaque pointer to the underlying cloud service handle. By default, this is the handle to an Azure Iot C PnP device client.
messageThe message content. For example, a JSON string containing 'reported' property of the IoT Hub Device Twin.
responseCallbackA callback to be called when the device received a http response. If NULL, the default callback will be used, which will stop processing a message if http status code is >= 200 and < 300.
completedCallbackA optional callback to be called when the messages processor stopped processing the message.
statusChangedCallbackA optional callback to be called when the messages status has changed.
userDataAn additional user data.
Returns
Returns true if message successfully added to the pending-messages queue.
